# ユーザー認証機能

## 概要

ユーザーがメールアドレスとパスワードでアカウント作成・ログインできる機能。
MVPとして最小限の認証フローを提供する。

## 背景・ADR参照

（関連ADRがあればここにリンク）

## 要件

### 機能要件

- MUST: メールアドレス + パスワードでのアカウント作成
- MUST: ログイン / ログアウト
- SHOULD: パスワードリセット
- COULD: ソーシャルログイン（Google, GitHub）

### 非機能要件

- パフォーマンス: ログインレスポンス 500ms以内
- セキュリティ: パスワードはハッシュ化して保存

## スコープ

### このフェーズで対応するもの

- メール + パスワード認証
- セッション管理

### 対応しないもの（後続フェーズ）

- ソーシャルログイン
- 二要素認証
