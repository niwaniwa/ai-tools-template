# ドキュメントライフサイクル

## ステータス定義

| ステータス | 意味 | AIツールの扱い |
| --- | --- | --- |
| `wip` | 作成中・レビュー待ち | 参照のみ可。実装の根拠にしない |
| `accepted` | 承認済み・有効 | 実装の根拠として使用可 |
| `archive` | 廃止・過去の記録 | 参照しない |

## 遷移ルール

```text
wip → accepted   : 人間のレビュー承認後。未解決事項がすべて解消済みであること
accepted → archive : 新しいドキュメントに置き換わった、または不要になった場合
wip → archive     : 検討の結果、採用しないと判断した場合
```

- ステータスの変更は**人間の明示的な指示**が必要
- AIツールが自動でステータスを変更してはならない

## frontmatter スキーマ

すべてのドキュメント（`_template.md`を除く）は以下のYAML frontmatterを持つ:

```yaml
---
id: "NNNN"           # 連番（例: "0001"）
title: ""             # ドキュメントタイトル
status: wip           # wip | accepted | archive
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

ドキュメント種別ごとの追加フィールドは各テンプレートを参照。

## AIツール向けルール

1. **実装時**: `status: accepted` のドキュメントのみを根拠とする
2. **wip**: 明示的に指示された場合のみ内容を参照してよい。実装の根拠にはしない
3. **archive**: 現在の作業では参照しない
4. **新規作成時**: 必ず `status: wip` で作成する
5. **矛盾検出時**: accepted ドキュメントの内容が現状と矛盾する場合、実装を進めず人間に報告する
